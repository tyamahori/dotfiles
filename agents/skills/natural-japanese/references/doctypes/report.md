# 調査レポート — 数字を読者の代わりに翻訳しきる

## この型が答える問い

調査レポートが答えるべき問いは「何が起きていて、次に何をすべきか」である。データを集めて記述するだけでは調査は完結しない。記述は手段にすぎない。読み手が知りたいのは数字の羅列ではなく、その数字が意思決定にどう関わるかであり、著者がその翻訳を代行しなければ、読み手は自分でグラフを睨んで解釈をやり直すことになる。良いレポートは、冒頭の数行を読んだだけで「結局どうすればいいか」まで判断できる。悪いレポートは、最後まで読んでもデータの記述で終わり、そこから先は読み手に委ねられている。

## 必須要素

冒頭サマリー(結論と主要数値)、目的とスコープ、前提・母集団の定義、本題(調査結果)、方法(第三者が再現できる手順)、データの限界、So What(次に何をすべきかの提案)。この7つのうち、方法とデータの限界は省略しない。分量は少なくていい。方法がなければ数字の信頼性を読み手が検証できず、限界がなければ数字が独り歩きする。

## 構成の型

サマリーピラミッドで組み立てる。冒頭に結論と主要数値を置き、本文でその結論を裏づける詳細を展開し、技術的な詳細(集計クエリ、除外条件の内訳など)は末尾か付録に回す。読み手は冒頭だけを読んで意思決定できる。根拠を確認したい読み手だけが本文以降に進む。続編の調査であれば、冒頭サマリーの直後に前回調査との関係を一段落で書く。何が変わって再調査したのか、前回の結論のどこを継承しどこを覆すのかを先に示しておくと、本文の位置づけが定まる。訂正すべき誤りが過去にあれば、末尾でなく冒頭近くで明示する。信頼は結論を読む前の段階で確保しておく必要がある。

## 品質基準

**冒頭だけで意思決定できる**。基準は単純だ。冒頭サマリーを読んだ人が「結局どうすればいいか」まで分かるかどうかで良し悪しを判定する。数値の羅列だけで終わり、それが何を意味するかが本文の奥にしか書かれていないレポートは、サマリーの体をなしていない。

**事実と考察が構造で分離されている**。「〜だった(出典: アクセス解析、7月集計)」という事実の記述と、「〜と考えられる」という著者の考察を、地の文の中でも見分けがつく形で書く。出典の一次・二次の別も明示する。自社の一次データなのか、他社事例からの類推なのかを混ぜて書かない。

**誤読しやすい数字は著者が先に指差している**。合計や平均が特定の外れ値に支配されている、母数が小さくて比率が不安定である、季節性で単純比較できない、といった落とし穴は、読み手が気づく前に著者が注記する。注記を末尾の但し書きに隔離せず、該当する数字のすぐそばに置く。冒頭の数字だけが一人歩きし、注意書きが誰にも読まれない構成は避ける。

**数字の意味が翻訳されている**。「継続率は4%だった」で終わらせず、「裏を返せば96%が離脱している」まで書く。読み手が電卓を叩かなくても意味が分かる状態を作る。

**母集団の境界が本文の早い段階で定義されている**。曖昧にしない。「訪問数」がボットを含む全アクセスなのか、除外後の実訪問なのか、集計期間はいつからいつまでかを明示する。この定義が調査のたびに揺れると、前回との比較そのものが成立しなくなる。

**セグメントごとに濃淡がある**。すべてのセグメント・すべての期間を同じ文型・同じ分量で並べない。異常や示唆のあるセグメントは厚く、平常のセグメントは「特に変化なし」の一言で流す。

## AIがやりがちな失敗

生成させたレポートは、データを記述するところで力尽き、So Whatまで踏み込まない失敗が最も目立つ。「訪問数は前月比で減少した」で文章が終わり、その先の「だから何をすべきか」が本文のどこにも書かれていない。もう一つの癖は、セグメント別の記述をすべて同じ文型で並べることだ。「Aチャネルは〜だった。Bチャネルは〜だった。Cチャネルは〜だった。」という三段跳びの反復が続くと、本当に重要な差分がどこにあるのか読み手には見えなくなる。注意書きを末尾にまとめて隔離するのも典型的な失敗で、冒頭の数字だけが独り歩きし、実は外れ値1件が合計を押し上げていたという事実が最後まで読んだ人にしか伝わらない。母集団の定義を明記しないまま数字だけ並べる失敗も多い。今回の「訪問数」と前回の「訪問数」が同じ定義で集計されているかを確認せずに増減を語ると、増減そのものが集計方法の違いを比較しているだけの可能性がある。

## 文体憲法の重点条項

- 第1条(結論から書く): 冒頭サマリーが数値つきの結論そのものであることが型の核
- 第12条(So Whatまで書く): 調査レポートは事実の記述で終わらせず、次の一手への接続を最低一段落書く
- 第11条(事実と意見を分ける): 出典・一次二次の区別と、考察部分の明示が直結する
- 第10条(限界・未確定の明示ラベル): データの限界と誤りの訂正を隠さず開示する
- 第7条(濃淡をつける): セグメントごとの厚みの差が読みやすさを左右する

## 雛形

架空のECサイト「そらマルシェ」の流入経路を調べた月次レポートの冒頭を例にする。

```
# そらマルシェ 流入経路調査(2026年6月分)

訂正: 前号(5月分)で「検索流入は横ばい」と書いたが、集計対象に
社内アクセスが混入しておりデータの誤りだった。除外後は前月比
5%減で、今号の「2カ月連続の減少」はこの訂正後の数値と接続する。

## 結論
検索流入の伸びが鈍化した一方、SNS経由の購入率がサイト平均の
2.3倍まで上がっている。次回施策はSNS導線の強化を優先する。

## 主要数値
- 総訪問数: 128,400(前月比 -3%)
- SNS経由の購入率: 4.6%(サイト平均2.0%の2.3倍)
- 検索流入: 71,200(前月比 -8%、前々月から2カ月連続の減少)

## 注記(先に断っておく)
SNS経由の購入率のうち、上位1商品(季節限定の虫よけスプレー)が
購入全体の41%を占める。この商品への言及が6月中旬にインフルエ
ンサーの投稿で急増したためで、SNS経由の高い購入率はこの1件に
かなり支えられている。この商品を除くと購入率は2.1%まで下がる。
```

## before/after: 記述で終わる文から So What まで書く文へ

**Before**

> 6月の総訪問数は128,400件で、前月から3%減少した。検索流入は71,200件で前月比8%減、SNS経由の購入率は4.6%だった。

**After**

> 検索流入は2カ月連続で減少しており、6月は前月比8%減の71,200件まで落ち込んだ。裏を返せば、全訪問数の減少3%のうち大半を検索流入の落ち込みが説明している。一方でSNS経由の購入率はサイト平均の2.3倍にあたる4.6%まで伸びた。ただし上位1商品への偏りが大きく、この商品を除くと2.1%まで下がる点は割り引いて見る必要がある。次回はSNS経由の導線をこの1商品に依存しない形で強化することを優先したい。

Beforeは数字を並べただけで、読み手は「それで、どうすればいいのか」を自分で考えなければならない。Afterは減少の主因を数字で説明し、伸びている数字の裏にある偏りを先回りで指摘したうえで、次の一手にまでつなげている。
